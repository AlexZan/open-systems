# Thoughts

How is content determined as owned by the OP or owned by the community?
Or rather editable by only the OP or by the community?

#### Conclusion

Any content can technically be voted to be modified, but it must first
be voted as open content. When a user makes a post, it will be default
proprietary. Users are unable to make suggestions such as edits if
content is set as proprietary. Although proprietary content can be voted
to be changed to open and vice versa. When content is marked as open,
anyone can make suggestions to any part of the content, and those
suggestions can be voted in.

If I post a reply to another user, should others be able to vote to
modify my message? No because my response is only for that user, it is
not information meant to value everyone. Although in some cases, as in
helping that user with a question, which might benefit others in the
future, users should be able to modify the response.

If I post a response to a QA list, should others be able to vote to
modify my response? Yes because the content is to value others.

If I post a biased opinion, should others be able to modify my response?
No, because it is my personal opinion, and it can not be objectively
proven wrong. If it can be objectively proven wrong, maybe it should
still not be modified because the dialog of stating the opinion and
having it proved wrong is the value for others.

So is the idea that if the content can value others, it should be
modifiable by the community? No because in the above example, the
dialogue can value the community, but it should stay intact. Because its
a form of evidence.

So then does it depend on the designation of the content? Or if the
content expresses an opinion, that should not be changed?

What about a typo to a response, should someone be able to fix that
mistake? Maybe no because the mistake makes it more authentic, and the
mistake could be part of the dialogue. It is closer to the truth of the
events.

Is the idea to maintain the truth of events then? If i post and I say
that aliens built the pyramids. Could someone change that? No because it
is my opinion, and the truth is the even of expressing my opinion, even
if my opinion is not accepted as correct.

When someone is posting a "page" or content that is meant for the
public, it should be maintained by the community, but when it is the
expression of opinion, it should not be modifiable. Should it not be
modifiable by anyone, including the OP?

If an OP was upset, posted something rude, then later calmed down and
apologized. Should the op be able to delete or modify the original
message? Or is the value in the dialogue with self or others? I think
the value is in the dialogue, going from wrong to right. The dialogue of
realizing a greater truth. The dialogue of expressing anger and then
regretting it. There is value in that.

So maybe when a user posts, by default they are posting an opinion, and
it is immutable. But if they create a public post, such as a goal or a
project, these things are community managed.

What if they wish to have a goal to raise funds for a non open company.
No one should be able to modify their "page" since it is for a private
project, company, goal.

### Thoughts on Canceling Upvote

Should users be able to upvote content and then cancel their upvote if a
post has not changed since their vote? No because an experience farmer
could vote in one direction, then cancel their upvote when they see it
trending in the opposite direction, just to get a return on their
investment, which is not value of an expert. An expert should be
incentivized to vote early before anyone else because they have a
certain confidence on the subject, they lock in their opinion and hope
that there is value in it and they get a return.

#### IDEA! DOUBLE MAX NEXT GOAL FUND

A previous goal must be completed with no less than half the value of
the next goals fund. If a goal of 40k is achieved, and the funds are to
be delivered, the previous goal must have been at least 20k, and have
been voted as completed. The goal before that must have been at least
10k, then 5, 2.5k, etc at some point the OP will have to front the value

as the community might not provide second chances if there was proven
dishonesty. Yes someone could try to scam, get caught and have their
account flagged, and try to make another account. But in order to create
another project, they will need a great deal of merit in the field of
their project, or else no one will take them seriously, and to gain
merit, it will take a great deal of work. Therefore scamming with an
account that has merit has costly consequences.

### Revisiting 0 day

When there is no exp in the network, but it requires exp to seed exp,
how will the first users gain exp? Auditing is a solution that provides
experience to new users by having them vote on a random new post. This
deals with the issue of a user creating multiple bot accounts, to just
vote on their main account to gain free experience. But even bots could
audit, and as long as some of them get some correct votes, they can gain
experience to vote with, and can still exploit. The solution is to set a
best of x, e.g you must vote on 5 consecutive audits in order to be able
to earn the experience. That way a bot must vote 5 times correctly, and
the odds are not there. If any vote is not "correct" the count is
restarted.

If a post is voted with a tag, and after 24 hours it has more borrowed
votes than actual votes, then that post is sent to auditing users of
that tag. E.g. a post is made with #electronics gets 100- votes but only
50 votes after 24 hours. This post is now sent to auditors. Once the
actual votes pass the borrowed votes, the experience is awarded to the
borrowed votes as long as all the other requirements are met.

### Who gains or loses experience for a public wiki/page?

### Tags network

A tag that is a child of another tag will provide 1/2 the experience to
the parent tag. A tag that is more specific is a child then the related
generalized tag. This hierarchy will be developed and maintained by the
community. E.g someone that has experience in capacitors should also
have experience in electronics, and the sciences.

### Borrowed Votes

Users without experience in a tag can still vote with up to one borrowed
post per post. A vote can not be changed or cancelled unless the post
changes. As the user gains actual experience from content contributions
or audits, their borrowed votes, if actualized will double.

### Up to Double Return on Invested Experience

When voting on content, you can gain up to double if you are the first
to vote on the post and it is actualized. The closer you are to the
first, the closer you are to double.

#### Examples

There are +1000 votes, and you vote +1, when it reaches 2000 votes, you
gain double? Since there is now the risk that it might never reach
double the votes.

### Revisiting Vote Threshold

The idea is that votes must pass a threshold in order for experience to
be awarded. This stops farmers from trying to gain experience from a
post that has already been established. Ideally a vote before this
threshold is done silently. So vote values are not actually submitted,
and can be changed. Only a signal is sent saying a vote has been made.
After 24 hours, the votes are revealed. If positive votes are not more
than double than negative, or vice versa, the vote does not pass the
threshold, and goes into silent voting again for 24 hours. This can
repeat indefinitely, but users will not want to spend any more
experience since a vote might never pass the threshold.

### DownVote Idea

If a post has double the down votes than upvotes +1, it can not be
downvoted anymore. Eg. a bad post gets two negative posts in that tag,
and has no upvotes, the downvote button is removed. If a post has 1000
upvotes, and 1000 downvotes, it can still be downvoted. Once it reaches
2000 downvotes, it can no longer be downvoted, unless the upvotes
continue to go up. The user would then receive -1000 experience.

The action for experience would be experience assignment.

When you vote on a post, you commit to verifying it. If you do not
verify the votes by the end of the voting period, you lose your staked
vote.

Use case:

Day 0:

User makes a post with #code tag. There are no downvotes after 1 day.
Users looking at the post after 24 hours each verify that there are no
downvotes. After a day, if there are no discrepancies in total
verifications, then exp is assigned.

0 views:

If there are no views, there is no one to execute the exp reward.

1+ views, but 0 votes:

There is a view, but no one has voted. By default 1 experience is
awarded without anyone voting or staking (this will eventually only be
available to users with less than 1 experience)

1 vote:

Verification can not happen until there is more than 1 vote.

2+ votes:

1 day after the second vote is received the verification period begins.
Both users launch the app and the verification is executed on their
clients. Optionally the users can run a dedicated peer software which
will automatically verify all the votes they are participating in. The
votes are verified. Once the verification period is over and there are
no discrepancies. The original poster can claim their exp reward, or if
they have the dedicated peer software, this is done automatically for
all of their posts.

1.  First vote made 5 hours after post

2.  Second vote made 10 hours after post

3.  1 day after post verification can begin

4.  The first voted user starts the app and the votes are verified and
    pass

5.  The second voted user starts the app and the votes are verified and
    pass

6.  Since the second voted user was the last, the verification period
    ends and the full verification passes

7.  The original poster launches the app and the reward of 2 exp is
    claimed.

Discrepancy in verification:

If there is a discrepancy at any point during the 1 day verification
period. The verification is redone across all dedicated peers even if
they are not subscribed to the post or had voted on it. The user(s) of
the flawed verification loses their staked experience. The process
continues as normal.
