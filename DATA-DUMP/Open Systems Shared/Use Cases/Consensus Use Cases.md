### Global proposal, no block, not time sensitive

1.  Create new post with proposal with a contract to execute the
    proposal

2.  Once more than 50% of users globally acknowledge that they have
    looked at the proposal, and there are no blocks, the proposal
    executes

### Global proposal, with block retracted, not time sensitive

1.  Create new post with proposal with a contract to execute the
    proposal

2.  A user blocks the proposal with a concern that it will break a sub
    system

3.  Acknowledging is disabled for the time being, at 20%

4.  Another user responds to the block, proving that the sub system will
    be fine

5.  The block is retracted by the owner

6.  Once more than 50% of users globally acknowledge that they have
    looked at the proposal, and there are no blocks, the proposal
    executes

### Global proposal, with block sustained, not time sensitive

1.  Create new post with proposal with a contract to execute the
    proposal

2.  A user blocks the proposal with a concern that it will break a sub
    system

3.  Acknowledging is disabled for the time being, at 20%

4.  Users upvote the block

5.  The owner responds to the block saying they see the issue and the
    proposal is retracted by the owner

6.  The owner can repost the fixed proposal and cross-link it to the old
    post.

### Global proposal, with block by troll, not time sensitive

1.  Create new post with proposal with a contract to execute the
    proposal

2.  A user blocks the proposal with a nonsensical reason

3.  Acknowledging is disabled for the time being, at 20%

4.  Users report the block as mischief

5.  Random users and auditors are asked to acknowledge the block is
    mischief, and that it should be disabled with the account reported.
    They all acknowledge and no one blocks the report.

6.  The block is disabled and the acknowledgement can continue.

7.  Anyone that subscribed to the post is notified of the continuation

8.  Once more than 50% of users globally acknowledge that they have
    looked at the proposal, and there are no blocks, the proposal
    executes

### Global proposal, with block by troll, report blocked, not time sensitive

1.  Create new post with proposal with a contract to execute the
    proposal

2.  A user blocks the proposal with a nonsensical reason

3.  Acknowledging is disabled for the time being, at 20%

4.  Users report the block as mischief

5.  Random users and auditors are asked to acknowledge the block is
    mischief, and that it should be disabled with the account reported.
    They all acknowledge and one random person blocks the report
    accidentally, with an invalid reason.

6.  The block on the report is reported as accidental

7.  Random users and auditors are asked to acknowledge the block is
    accidental, and that it should be disabled. They all acknowledge and
    no one blocks.

8.  The last block is disabled and the acknowledgement can continue.

9.  It passes and the report goes through

10. The first block is disabled and the acknowledgement can continue.

11. Anyone that subscribed to the post is notified of the continuation

12. Once more than 50% of users globally acknowledge that they have
    looked at the proposal, and there are no blocks, the proposal
    executes

### Global proposal,with no block, and time sensitive

1.  Create new post with proposal with a contract to execute the
    proposal

2.  As consensus is occurring, users realize this is a time sensitive
    proposal (severe bug fix) and they begin to upvote the tags
    supporting the decision.

3.  As soon as all viable tags reach majority upvote of global
    experience, the proposal is executed.
