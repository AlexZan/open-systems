# Use Cases

## My friend was injured, hospital bill was xx, can we raise it?

In this case there is no bounty, there is no goal, any amount helps.
Ideally proof of help needed would help raise funds, such as the bill
itself, pictures of the friend. Proof of payment afterwards would help
raise karma. More karma if their friend posts recovery follow ups to
show how the communities contribution helped as well as thank you's.

## I have this idea, i would like to turn into a business, Can u help me raise xx for a prototype

In this case the user is asking to retain ownership for the production
of a business, at minimal the ownership of the prototype, but most
likely this will lead to requests to raise money for full production and
thus to retain ownership for the full business.

Instead they could create a goal and bounty for creating the prototype
to a suggested specification. This way the community can help improve
the specification, as well as improve the speed and quality of the
prototype development.

This would be suggested, and a goal and bounty could be started.

The original poster will be credited for the concept, and any
contributions they made. They can even be the one that delivers the
prototype and receives the bounty. This will create a healthy
competition between any users making the prototype, or will result in
collaborations.

A user could post a small progress on the prototype, users would see
value in this and vote shares for it. Another user would use that to
build the next part, and so on.

## A big company is using my copyright artwork, i don't know what to do

In this case there is no initial bounty or goal. The community might
vote in a new bounty to the existing post setting a goal of having a
lawyer or firm take legal action and resolve the case. The community
could even create a second bounty, for winning the case, which would act
like a bonus to the firm that takes the case.

law firms or lawyers can then bid to represent the user if he properly
states their case, and has a good case. If the bounty grows to a
sizeable amount, and there are several bids from firms. Then the
community will vote on a bid.

If there are new developments that require further action or funding,
they can be posted as sub posts to the main posts with sub bounties and
goals.

In this case certain details can not be public, as they might provide an
advantage to potential defenders looking for exploitable information,
and so specific bounties that might require this information to be
completed can only be completed once the case has ended. Therefore in
certain cases, hold times and completed times for bounties could be
long.

## A user copied another user\'s work and submitted it as their own

In this case there is no goal or bounty. The claimant will add any
supporting evidence they have. Any other user can add their own evidence
to support or deny the claims made.

The community will then vote for or against the defender

The community will begin to post resolution options once a guilty vote
is finalized and then vote on the resolutions. E.g.

1.  Pay Damages (take to local court if no damages paid by x)

2.  Remove xx karma from user

3.  Ban user for xx time

4.  Public Apology and essay on wrong doing from guilty party

## A traffic light advanced green seems broken

This post specific to a location.It will have a bounty to fix the
problem. People from that area are likely to bounty donate and post.
Although anyone globally can donate and post if they like. The bounty
will be distributed to those that made progress. Users can vote as a sub
post making progress with the + button.

## I have this great idea for an app, I want to make it into a business I own

This post will have goals and bounties. For this type of post to gain
funding the user will have to provide valid reasons why they want to
make it an owned business vs an OW service that is community built.

Users might try to explain the benefits to the user of having the idea
as an ow service, that they will still be compensated.

Case: Greed or Insecurity

The user might express an interest in being compensated for their
concept and any existing work. If the community agrees to this they will
create a funding goa as a bounty which both the community and the user
aggress to. They will also agree to the users share in the project, they
might do less or more depending on initial monetary compensation. Once
the goal is achieved. The terms of the goal from both sides must be met
before the community votes as the agreement achieved. The user will make
all his work public within any agreed systems within OW, such as an open
system for source code, documentation, etc.

Case: Forceful clone

OW users can just copy his idea and develop it as a community service.
Since the original poster has proof they posted about the concept first,
they can request that this service be terminated, if it does not
terminate they can create a intellectual theft cause against those who
continue to progress it.

Case: Agrees to make it Open

After discussion in the post about making it an open service, the user
sees the value in doing so, and hopes that his work is compensated for
through donations. He makes his work public under the various OW
systems. The community creates a funding goal on its own to compensate
the user, and donations and sub bounties to the project as it gets
created and finally used. As the project is used it continues to
generate revenue for the user and contributors based on share assignment

Case: Overrules and wants it private

For whatever reason, the user still wants the business private. In this
case they can create a post with a fund request, and hope that users
will donate to the post. All the donations and withdrawals are made
public. The user can post a reason for the purpose of their withdrawal
and what they are going to use it for, as well as follow up with the
result of using that funding. This is likely to encourage further
donations.

## Lets design the best smartphone we can with the available tech for this year, that we can then manufacture and ship globally

This initially will have a bounty for the goal which the title
describes. But as it gains traction it will probably have some community
added sub goals such as

#### Vote on requirements

Here the community will add questions, the questions with most likes
will get shown at the top first.

> What resolution interests you? 1080p,4k,8k?
>
> What size dimensions would you like?
>
> What size bezel, or do you prefer edge to edge?
>
> What Size memory?
>
> In screen fingerprint scanner?
>
> What parts would you like modular? Can be removed or upgraded

#### Best Concept Rendering Bounty

Now that the requirements are in 3d designers can begin to submit their
renderings of what they think it could look like.

#### Best schematics Bounty

Now that a design is selected, an electronic/mechanical schematic can be
voted

#### Bids for Manufacturing and Delivery

At this point a Beta Bounty in each region can be started. This bounty
is a bit special in that, you are not donating, rather reserving a beta
unit with the set amount u contribute to the bounty. E.g. \$300 beta
cost.

When the beta bounty in a region reaches a certain point, manufactures
or brokers will start bidding for the work. The community will vote on a
manufacturer.

Since this is a large project, a manufacturer will be expected to
provide milestones that the bounty will break down into with quality
assurance every step of the way from presenting their progress to the
community.

#### Beta Feedback

Once enough feedback from users comes in, some suggestions might drive
bounties in order to incorporate them into the version 1.0. These will
follow the same process as above with specification, concept, schematic,
manufacturing.

Now a new bounty for manufacturing and delivery of version one is
started, people will pay \$500 in exchange for a reservation. Once the
bounty is high enough, bids will be taken and voted on. Phones will
arrive, feedback will occur and there might be a version 2.0, or a
version 2020 from that feedback.

## Contributors should get a percentage of service transaction cost

"The amount would be community voted. E.g. the contributors that made
open 'uber' will get 5% of each service transaction cost (2.5% from
rider, 2.5% from driver), or the price of an uber ride. This can be
overwritten by the end user when they pay for their ride, or by the
driver when they receive their pay."

This would most likely require a bounty so that a programmer can
implement this new functionality. It would be general functionality for
all service contracts that have a fee.

When the bounty hits a goal, an interested developer will post that they
are working on it. Then they will post a git repository with the code to
the completed version. It will receive initial feedback and suggestions
or fixes from other developers and these developers will also become
contributors and receive some share.

The community will decide to accept the bounty goal as completed before
or after the project is put on staging or not, but at some point it will
get to staging. Here it will be tested by users and more feedback will
be posted. The more helpful the feedback that leads to an improvement,
the more of a share in contribution the user will receive by the
community.

When it is finally time to go live, a community vote post is made to the
original post "should it go live?" The vote passes and a smart contract
deploys the project smart contract from staging to live.

### Tech

Since all the code will be open source this can be a public repository
of code on the blockchain. This can follow a similar pattern to a
transaction record but for code. When a smart contract votes to push
changes from another repo such as staging or a post repo, it will
propagate to all the nodes that hold the live repo. OW functions
directly from this live repo. Malicious users may try to alter their
node, but the 51% rule applies. Any change that needs to be done to the
live code of OW must be community voted in. This vote activates a
"delivery" smart contract which takes a source repo and pushes it into
the live.
