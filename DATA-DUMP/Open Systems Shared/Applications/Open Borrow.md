[A system where people can borrow things for other things using specific
tokens.]{.mark}

[e.g. A user lends out wheel barrel for a day. Based off community set
data, the wheel barrel has a value of 10 tool tokens per hour. The user
earns 10 x 24 tokens, or 240 tool tokens. They can now borrow a
chainsaw, for 20 tool tokens an hour from a neighbor.]{.mark}

# Assessment, Damage Claims and Disputes

[When an item is borrowed, both users must checkoff that it is
functional and has no major damage. If both the user lending and the
user borrowing the item checkoff, then it is transferred. Once again
when the item is returned, the same transfer process takes place. If
both users sign off, then it is transferred back and returned to its
owner. If both users dont sign off, a damage claim is generated. If a
user accepts fault, they must cover the repair and any other damages. If
they don\'t accept fault. They must dispute it in an open form and
provide supporting evidence for their dispute.]{.mark}

# Late fees and no returns

[A user will continue to get charged the hourly rate for as long as they
have it. If they pass the allowed time by the owner, this rate doubles,
unless an extension is requested and accepted. The user set a no return
cut off, at which point the item and borrowing user are market as no
return. The total value plus 50% of the item must be payed to the
lending user by a set time. If the payout is not delivered, Open Trade
will pay the lender the total amount from the insurance pool and will
begin enforcement proceedings on the no-return user.]{.mark}

# Enforcement and Insurance

[If a no-return user does not pay out what they owe by the due date,
several enforcement steps will take place. Collections and eventually a
court summons. All overhead costs must be payed back to open trade by
the no-return user. Prior to collections, Open Trade will pay the
lending user the amount they are owed out of an insurance pool. The
insurance pool is kept healthy by a % overhead cost from users
purchasing tokens with cash.]{.mark}

# Account, Reviews and History

A user will have an account with a history of all they borrowed and lent
out. Reviews from other users is optional. Any no-returns will show
here. An active no-return will mark the account as suspended.

# Debatable

## Should tokens be tradeable?

[By making them tradable, exchanges will then be driven by profit, not
utility creating an overhead in regulation, security and insurance. e.g.
A user lends out wheel barrel for a day. Based off community set data,
the wheel barrel has a value of 10 tokens per hour. The user earns 10 x
24 tokens, or 240 tokens. They can now borrow a chainsaw, for 20 tokens
an hour from a neighbour. If they were able to trade these tokens on an
exchange, it could then become a business of renting out tools. The
incentive would then become profit. Without tradeable tokens, the
incentive is simply to borrow tools. This needs to be debated.]{.mark}

## How specific should tokens be?

[If they are too generic, the token will become commercialized once
again. e.g. I will keep renting out tools so that i can earn tokens in
order to borrow cars each day.]{.mark}
