## Challenge: Voting & Authority

If; decentralization removes central authority and democracy requires
voting. The question is then, how is voting achieved with no central
authority?

First let's look at how voting is achieved with a central authority.

The ballot is cast by the individual. Central authority then verifies
the identity of the voter, to make sure they are permitted to vote. The
main consideration for being permitted, is that they haven't already
voted, so that they don't get multiple votes. Other considerations could
be, false identity, or not being in the jurisdiction.

How can a decentralized system then be responsible for those things?

Starting with identifying the voter:

Maybe the user can submit their government id to the blockchain, but
then what authority will assess and approve the id. The other issue with
this is that it is dependent on central authority; the government,
negating the main objective.

So then how can a user identify themselves without government issued
identification. There are many complicated technological and logistical
solutions.

But let\'s take a step back first, and see how well traditional
identification for voting currently works. If you head over to
voterfraudfacts.com you can see various statistics that show it fails
regularly. With the current system there is no way to guarantee fair
voting, because there is no way to guarantee identity.

### A Radical Proposal; Remove the Need for Identity

If a user does not need to identify, then there does not an authority or
an enforcement for that identity with voting.

The first question is, does that not make voting multiple times an even
bigger issue? A user could just keep creating multiple accounts to vote
with and get indefinite votes.

It is not a problem if you allow it. A user can vote as many times as
they like. How will voting then be fair?

Voting power will then not be based on solely the individual or user.

Each user or user account will initially start with 0 power. They are
not able to influence the system in any way. The only thing they can do
is contribute content. Such as, posting ideas, running campaigns,
starting discussions, replying. The community then assesses the value of
each content contribution, and votes on its value. This value then
directly translates into voting power for the user. This is an abstract
explanation of the experience and voting system.

A user could make as many accounts as they like, since it is their
individual effort alone that will give them voting power, so whether
they choose to distribute that effort across multiple accounts or focus
it on one, it is their choice.
