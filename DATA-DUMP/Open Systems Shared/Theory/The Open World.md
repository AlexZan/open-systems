The Open World is a set of applications and services based off Open
Systems Principals.

This document extends the Open World Introduction and brings in more
details and examples of its functionality.

# Open Posts

An open post is owned by no one, but anyone can attempt to contribute to
it

*\*\*what about incentives and donations to open posts, should the
original posters not get something? \**\*

Open posts are written to the blockchain, and so it will cost a tiny bit
of crypto in order to post it. Writing to the blockchain provides an
immutable record of all changes made.

Anyone can propose changes to an open post. This list of proposals is
available to all who choose to see them. These proposals can be sorted
by the tags they are voted in. A user skilled in grammar can sort by
#grammar and propose changes, earning more experience in #grammer if the
proposals are selected. The user will also earn any royalties from
successful projects for any contributions they made: They will earn a
small percentage of royalties for successful grammatical edits.

## Edit Proposals

If there is an issue with an open post, or if it can be improved, users
can submit edit proposals. Edit proposals are kept in a list and not
shown by default, as not to impede in the default post experience. A
user must choose to view edits. An edit can be a change to any part of
the post, text, image, audio, etc. Viewing the edit will showcase the
proposed changes so that they are noticeable against the original post.

In order for an edit to be accepted, it must maintain a greater amount
of votes than the last edit for 1 day.

When an edit receives greater than current votes, all users viewing the
post will see a notification that the post will be updated within a day.
They can then counter vote if they disagree.

E.g. a new post has a grammatical error that a user fixes. It gets a few
#grammer votes within a day. Since it is a new document, there are 0
edits with #grammer already and so a few votes in #grammer are greater
than 0 and the document updates with that fix. Later another user
notices a historical fact error, posts the fix with a comment about
their reference, other users vote #history and a day later, the update
gets posted.

# Open Projects

## Example: Just an idea

A user has an idea for an app, but they do not have any skill to design
it, develop it, market it or operate it. Instead of keeping the idea to
themselves, they post it on Open Forums. Let's say it\'s a tool sharing
app. If it\'s a good idea, the app might start to trend in upvotes for
#app idea and some #tools votes. It gets enough upvotes and exposure and
replies that a user replies with an open project post. This open project
post is a stand alone post, but it also behaves as a reply to the
original idea post, to allow other users to easily find the project
post. The project post also references the original post to provide
credit to the original poster for the idea.

The project post proposes various goals, such as design, visual mockup,
prototype, beta. The community begins to reply and shape the project
goals.

Skip forward to the project finished and operating, with many users
enjoying the benefits of the tool sharing app and service, the user with
the idea enjoys a monthly 5% royaltee cut just for the idea, as long as
the app earns revenue, and as long as no one votes against the user.

## Profit Share

When a project begins to make profit, the profit is automatically
distributed to all contributors based on the amount of value they
contributed.

E.g. A tech open project received multiple replies with #engineering,
#programmer, #design, #funny, #grammer, and about a dozen more, during
development.

During the lifetime of an open project, and prior to profits being
distributed, tags must be voted in by significance to the project. These
are the tags that users will decide should get the greatest profit
share.

In the above example, since it is a tech project, the tags #engineering,
#programming get the most votes for profit share, but also #grammer gets
some since it did contribute to the overall project.

Profits are distributed to users who contributed the most in the most
significant subjects; users of posts which received the most upvotes in
subjects that were upvoted the most for profit share.

## Profit

When a user decides to pay for a service, the crypto payment goes
directly to a policy smart contract which auto distributes the payment.
In some cases the funds are distributed immediately, in some cases they
could be distributed monthly, depending on the selected policy.

All users who earn profit share will receive their part after a purchase
is made.

# Voting

## Vote Comment

Whenever a user votes, they also have the option of supplying a message,
typically used to describe the reason for their vote. This message
behaves just like any other post, and users can discuss the reasoning.
This can help form a more educated consensus.

## Negative Votes

If a post has double the down votes than upvotes +1, it can not be
downvoted anymore.

E.g. If a new post gets two negative posts in that tag, and has no
upvotes, it can no longer be downvoted.

E.g. If a post has 1000 upvotes, and 1000 downvotes, it can still be
downvoted. Once it reaches 2000 downvotes, it can no longer be
downvoted, unless the upvotes continue to go up. The user would then
receive -1000 experience.

# Accounts

## Public Experience and Record

All experience is public. Users can view any other user\'s experience
and a record of the interactions which led to that experience.

E.g a user starts a private electronics project for funding. They have
1000 points in #electronics but also have -500 in ethics and -20 in
business. The users can view what posts or events they lost and gained
their experience on, and decide for themselves if they trust this user
with their funds.
