Consensus groups or Jury Audit Consensus is the joint use of a random
jury from the members, as well as auditors who have explicitly signed up
to audit. The group is an ad hoc representative non-elected body that is
used to make a time sensitive decision. The group is used only for the
single decision and is dissolved after the decision is made.

A jury member will get no penalty for wrong assessment, whereas an
auditor will lose the experience they voted with. Both jury and auditors
will gain experience for correct assessment.

Correct and wrong assessment is deemed by the end result. E.g. a post
gets reported for plagiarism and an auditor votes it was not, but it is
proven that it was, the auditor will lose the experience they spent on
the vote.

An auditor must sign up for the subjects they have experience in and
wish to audit. Reporting for example will have default reportable
offenses that have been democratically added. Each report type will have
its own preset subject types. E.g. plagiarism will have #investigation
as a default tag, and any auditor with #investigation experience can
vote on the validity of the report by looking at the evidence provided
and available.

The first x amount of auditors/jury up to a limit will dictate the
consensus.

## Time Sensitive

The only time to use a consensus group is when there is a time sensitive
decision where consensus would take too long. The benefit to a consensus
group is that if a decision needs to be deliberated, any group member
can block the decision; opening up to deliberation. If the matter is
urgent, deliberation can still be bypassed by a mass vote to execute it,
which would cost all the voters experience.

## Bad vs Good

This is subjective, thus these concepts work well with the consensus
mechanic in open systems. A bad decision is one that caused harm in
retrospect as decided by consensus. A good decision is one that creates
value as decided by consensus.

## Proposal vs Decision

A proposal is an idea for change, a decision is the outcome of whether
that proposal is executed or not. A proposal must be decided on in order
to be executed. E.g. A report is a proposal in order to execute some
action to moderate content/accounts through consensus. The report must
be decided through consensus in order to execute the action associated
with the report based on the established rules.

## Rules vs Reports

Rules are proposed and established through consensus. If a user breaks
an established rule, a user may report them. Rules should have
established consequences also specified and approved by consensus. When
a report is validated through consensus, the consequence action of that
rule will be executed.
