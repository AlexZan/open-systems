Example:

A developer contributes to a project and they are given \$1000 dollars
equivalent in debit tokens (\$100 per token). When the project begins
earning revenue, the org sets debit tokens repayment to 5% of profit. If
the profit is 10,000 dollars, and there are 100 users with debit tokens,
they each are able to get \$100 dollars worth of profit by exchanging 10
tokens. Each profit reporting period they might be able to exchange some
debit tokens until they no longer have any.

Maybe, the value of debit tokens can be given by owner token holders.
Each time period, an owner token gets a debit token. The owner of the
debit token can either claim these themselves or issue them as a reward
as well (optionally). This mechanic allows the owners, who funded the
project, to issue debit tokens, to anyone that contributed to its
development, with ideas, code

\... any content. The more value that their content contributed, the
more compelled the owner community to provide them with debit tokens.
This incentivizes more open source development from the content
community.

Optionally, the owner of the debit token can not redeem the debit token
themselves. This keeps out investors who have no interest in the
project, but instead are trying to only profit. At the same time this
keeps out potential funding.

Optionally the policy can be specified that these tokens do not
guarantee any value; they are not assets. They are products of a
successful profit model, but that is never guaranteed. (as to abide by
any local trade/financial laws)
