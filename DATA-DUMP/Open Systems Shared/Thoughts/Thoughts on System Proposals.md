A system proposal is when a change in the system is proposed and then
voted on being integrated. E.g. a smart contract that will replace an
existing system smart contract. It is first tested on a test net, then a
vote must be taken for it to be integrated into the main net.

Just like any other vote the criteria must be previously established,
then after 24 hours if all voting criteria is double the down votes, it
passes. When the system proposal vote passes, the smart contract address
for the system proposal gets sent to a system proposal proxy, which
updates to the new address. Now the system will use the new smart
contract that was voted in.

How long should it take? Systems its a system wide change, changes are
critical, validation must be exhaustive.

1)  The proposal is made

2)  Vote is made on the staging duration

3)  At the end of the staging duration, if it passes, it gets
    integrated.
