# Locking Experience for Passive Income

Should users need to lock their experience in order to earn funds in it?
Or should it be earned on all non locked experience?

Case:

1.  User has 10 experience in #e

2.  User will maximise passive income and lock all 10 experience in #e

3.  A vote they are passionate about comes up and they want to vote 5,
    they must first unlock 5 from their passive income lock. If this is
    immediate, then this ads nothing but micromanagement. If there is a
    delay, such as a delay, they might miss the vote.

**At this point i see no benefit in locking experience for passive
income, and all non locked experience should earn passive income**

# Different types of funds

All funds must have an exit behaviour defined before they are created.

1.  Fund: basic fund with exit limit requirement on amount or time. A
    beneficiary is also required.

2.  Goal: like a fund, but with the beneficiary only receiving the funds
    if a defined goal is voted as being achieved.

3.  Bounties: same as goal but with anyone being able to complete the
    goal and the winner becomes the beneficiary.

4.  Project: A set of goals and/or bounties.

## The argument against goals

When will there be a need to create an objective fund that goes to a
specific beneficiary? Are bounties or projects always the better
alternatives?

Projects encourage goal creators to dissect their goal into multiple
goals, building funds for trust. While bounties allow for competition of
the objective.
