The concept that each token provides one vote.

## example

If a commercial project sells tokens, which will be redeemed for the
product or service, having 1 token provides one vote. When it is time to
accept a goal, the users who have purchased into the project with tokens
get to vote, the more a user is invested in the project, the greater a
vote they get.

Can this be combined with open democracy, maybe weighted, or competing,
or auditing? Auditing where experts must verify the votes.

Maybe this can be a precursor to open democracy.

Maybe this is just for commercial projects, token based projects.

What if you need 100 tokens to redeem the product, that way people can
just buy a few tokens to support the project initially, and as it
progresses they can purchase more as confidence rises, tokens can also
be traded.

Without open democracy, this will again create the influence incentive.

Could this also work for non redeemable tokens? So that with a donation,
you get a token, and you get nothing for that token other than voting
rights. That way those who direct the project are those who are vested
in it, supporting it. Anyone can still discuss direction, but only those
vested can direct.

Then experience can weigh or interact in some way with token voting.

Although this should never be used in non commercial projects, since it
would go against being able to buy votes.

Maybe this can be an option for commercial projects, but not required.

Should someone without experience have a greater impact on a commercial
project? Yes, just like in a claim resolution, the parties directly
involved in the resolution will have the initial absolute say on
accepting the resolution, only if either of them disagree, then it will
become a community vote. But also no, because in a claim resolution,
only the parties voting are those effected, but in a project, the
decision affects anyone with an interest, or maybe only those with an
invested interest?

Then the question becomes, does a project affect anyone, those with an
interest, subscribers, donators or only the invested?

The more vested you are in a project, the more say you should have.

Who should decide this? Would this work in an open project? Can an open
project be a token project? Yes, there is no reason not to secure a
stake in an open project for contributing funds. In that case what makes
the project commercial? An open project is intrinsically non profit.

## Case: electric car open project

1.  Discussion

2.  Open project proposal

3.  Goal proposals

At this point users can purely donate just to see the open source design
completed, with no vested interest. Or it can be proposed that users can
buy tokens, but since it is an open source project, who would they
redeem the token with? Maybe they are non redeemable tokens, but purely
voting tokens.

In this case it makes no sense that by paying more, you have more
influence over decisions.

But the open source design will be responsible for your eventual product
outcome?

Another option could be to fork it, and fund the fork in the direction
you want in a close project.

Since the project is open source, it can potentially affect anyone,
therefore anyone should be able to vote based on their subject
experience; open democracy.

## Case: production of open source car design

In this case the open source project was completed, and now users want
to locally produce the car. If it is an open project, then Decisions
such as selecting a manufacturer. If it is a closed project then the
owner will be selecting the manufacturer and most details.

Since decisions affect other people, maybe nothing but experience should
have any consequence on the vote. Funding and wealth should not be a
factor in a decision that affects other people.

If you are paying for a product, decisions about the product should be
yours and not a user who is not paying. Anyone can advise, and the more
experience they have the more they are heard, but decisions should
ultimately be with those that pay for the product.

Closed project:

1.  Owner selects open source design

2.  Users begin buy tokens

3.  Goals:

    a.  Prototype video (vote approve)

    b.  First phase (100 vehicles), all token holders vote, while those
        who received the vehicles write reviews.

    c.  Second phase (1000 vehicles), all token holders vote, while
        those who received the vehicles write reviews.

    d.  Can continue this way, value of token will continue to increase

In this case no one but those who are getting the product should be able
to vote, by the amount they have invested in the project.

In this case should subject experience even have a bearing? If someone
does not like their product, they should not have to explain why they do
not like it, they can just vote no based on their feeling.
