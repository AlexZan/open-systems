Open content is for collaboration. It can be like a wiki page for
factual content with references, or it can be an initiative such as a
bounty.

An open post is similar to a closed post, where a user posts some
content, but instead of just being able to reply to it, you can make
suggestions on existing or new content in the actual post.

E.g. a open post on WW2 as a wiki. Users can submit text, pictures,
videos etc. The suggested content can then be voted in by other users
with experience in those subjects.

When a user suggests new content to an open post, or votes on existing
suggestions they are now subscribed to the section of that suggestion so
they are notified whenever a change is made through vote or a new
suggestion is made so that they can vote.

E.g. An incorrect fact for Dday is discovered in the existing content
for the d-day section. A suggestion is made with a correction and the
reference supporting it. Anyone that had voted in the previous content
of the d-day section now gets notified to vote on the correction.

As usual voting acts as an investment and is a good way to gain
experience in a subject.

E.g by voting 10 in favour of the new d-day content, once it passes
(same approval mechanic as bounty) the voter doubles their exp to 20 in
#d-day. The

Each section of a post will have a list of suggestions. Sections are
created dynamically. Each sentence or point is its own section. All
content behaves as a suggestion, even the original content by the
original poster. If a suggestion gets downvoted to the max downvotes, it
is no longer visible by default. The suggestion with the highest will
votes will always be displayed by default.

A vote can not be changed unless a new suggestion is made. At that
point, votes can be cancelled, and made on any other suggestion. A vote
can not be made on more than one suggestion of the same section since
they are competing.

Return on votes will be the same as all other content, when the net
amount of votes doubles in the same direction from the point of a user
voting, that user now doubles their vote count.

### Bounty

A bounty is also an open post, since it is a collaborative effort that
anyone can contribute to it must naturally be open, it must only be
open.

A bounty will have a title and description. These will have a list of
suggestions from users, and from the original poster.

Use case: a user can make a bounty, and have one incorrect point in the
description. That sentence gets downvoted, and when it reaches its max
negative votes, it no longer shows. If a suggestion is made in its
place, and gets upvotes, it will replace it. If no suggestion is made in
its place or no suggestion gets upvoted, it will remain blank.

### Suggestions accepting

A suggestion must maintain the most votes for 24 hours in order to be
accepted; it then becomes the content that is displayed by default in
that section, whether it is blank or has content.

### Suggestion in new section

If a user wants to suggest a new sentence, this will create a new
section. Since it is new, there are no competing suggestions in this
section. If a suggestion maintains positive votes and no other competing
suggestions, by default it will maintain the most positive votes ( a
suggestion must have at least 1 vote in order for it to pass), and in 24
hours it will pass. At this point the suggestion will be visible by
default. If users deem it of no value, they will offdownvote it, and
after 24 hours if it has max negative votes, it will be removed.

#### Thought: A form of punishment for no value farming

The original user must have a net negative from posting something
negative. Therefore any experience they got in return, should now act
negatively. So if they voted 2 votes on their own suggestion, and got 4
in return once it passed, then they should have -8 experience if the
vote is reversed after it passes.

After more thought, this could be a case of overdesign. Since by voting
for your own post, you are already risking losing those votes, if you
are downvoted, so the risk mechanic is already in place, no need to add
another.

### Forking

If a suggestion for open content, such as a bounty, is so different than
the original, it might be best to create a new instance of the open
content. E.g. a wiki that has competing views (they can cross link)

### Tags

What tags will suggestions use to be voted. The idea is to choose the
tags that demonstrate the proposed value of the content.

If a user simply corrects a grammatical error in their suggestion, they
can tag #editing and #grammer.

Suggestion to suggestions

A suggestion can be made on a suggestion. E.g a suggestion might be
factually correct, but might have a grammatical error, and so suggesting
the grammatical fix should not compete on the same level as the factual
suggestion, or against all the other factual suggestions. The
grammatical suggestion will be specific to the single factual
suggestion.

What about two grammatical suggestions that don\'t complete, but
complement each other, or are both needed? Default voting behavior puts
them against each other. Then the first user to combine them in a
competing suggestion will get the vote.

What about two factual suggestions that don\'t compete, but again
complement each other. Since each sentence is it\'s own section, more
than one statement shouldn\'t be made per section, and thus there
shouldn\'t be complementary facts, or more than one per statement or
section.
