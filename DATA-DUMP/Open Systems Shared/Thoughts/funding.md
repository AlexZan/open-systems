# Types of funding

1.  I need x dollars to create x deliverable, by y date

2.  If I make x deliverable I need x dollars

3.  There is x dollars in funds, who can make x deliverable?

4.  It would be nice to have x deliverable, lets raise funds

# Deliverables

This can be a single goal, or multiple as milestones.

# Example Post: a p2p open taxi service

A concept with parallels to Uber, but with some critical improvements.

1.  User posts their location and desired destination.

2.  Drivers then bid on rides

    a.  A drivers bid will be automatic based on several factors, but a
        manual bid can be made instead.

3.  User can select a bid as they come in, or wait for a potentially
    better bid

4.  As an open system, there is no owner, and no profit. All funds
    exchanged are peer to peer. Funds from user go fully to driver.

5.  Any user can become a driver automatically, as long as they mark
    themselves as open to driving. They will get a notification for a
    bid option.

Users will continue to add and vote in features they would like. Each
feature is itself a post with a goal. Users will also fund the bounties
for each feature. As the bounties grow, developers will mark the goal as
being worked on. Users can then see how many developers are working on
the feature, also how long other developers have been working on it for,
this way they can choose to compete for completing the feature or not.
If for some reason a feature has its bounty being funded, but the
feature begins to receive downvotes, because maybe it is deemed a bad
feature, once the downvotes reach a threshold, the bounty will be
dissolved and the funds returned to their source.
