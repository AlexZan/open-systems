Maybe there is no "global" open system. And any group, nation, community
can start an open system. This will keep user experience separate.

The same individual can have the same type of experience in different
systems. Maybe individual open systems can reach consensus to subscribe
to another system. The one with the most subscriptions can be the
elected "global" system. Subscribing allows experience from the
subscribed system to be used in the sub system.

## Should the parent need to accept the child, or the child can select whatever parent it wants?

Since it would create a lot of overhead(?) for the parent to select all
children, maybe a child can set any parent. Would this affect the parent
in any way? If this is known then there is no explicit association and
thus responsibility to the child.

The only thing the child gets by selecting a parent is that they allow
experience from users in that parent to be used in their sub.

## Are all subs members only?

Seems so, but the rules for joining and governance must be established
before joining a parent.

## Why would two open systems join?

The only purpose is to share experience.

## Can the same user exist across multiple systems?

If a user is just a public token, then that public token will be unique
on any OS; therefore the same user can exist across multiple systems.

## Its not a parent and child model

If two systems can be parented to each other the parent child
model/analogy does not fit. Any system should be able to decide to allow
any other system to transfer experience into it. Since it is the one
systems decision, it should not affect the other system in any way,
therefor there should be no default reciprocity. If both systems decide
to allow experience flow to each other then that is explicit
reciprocity.

Some models that work better are waterfalls, since they are one way.
Things can go down, but not back up. But waterfall also implies a
multistep process like the waterfall development method which might add
confusion. Maybe valves. It needs a name that is a verb also.

Piped, linked, hooked.. Maybe subscribe!
