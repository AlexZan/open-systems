# **Ai coding \- spec**

Spec is the new source of truth. The process of turning an idea in your mind into a spec, we can call specifying. This involves getting your vision from inside your mind out and into spec files. This can involve the initial description, but usually between that and the functional product the ai will make many assumptions. And then you can iterate through a process of refinement that grows the spec and zeros the functional product into your idea. But that process can be expensive. Instead you can similate with mocks, use cases or expand the spec with q\&a. The closer a spec resemble your vision the cheaper and significantly quicker reaching the finished product will be. 

# **Ai project tools**

Git based project planner. Scrum/kanban that is just text based so that ai can read and manipulate is early. Part of the git repo so changes can utilize the decentralized approoval  process.

In the agent file can have an instruction to update the planner with progress, references.

At the same time can have a thought map, which maps all your md files together Linking them and showing the links visually. Allow any coding agent to update this in real time since it's a text based core. 

This can then link to your planer board, and you can see which ideas are linked to finished features allowing you to filter or focus whats left.

This should also link to your project documentation structure/hierarchy so that AI can as efficiently as possible build context.

Changing the parent child link in the map will change the file in the directory. A directory watcher can also update the map if a file is manually changed or added to a directory. 

Can expand the md file from the title to the summary to the fist page and so on. 

The md file can be edited in place.
